## Weekly Goals App - Technical Resolution

### Problem Solved
- Built weekly goals app for Bart but had Supabase integration issues
- **Root cause**: `amber_state` table has check constraint on `type` field - only specific values allowed
- `weekly_goals` type was rejected, had to use existing `creation` type instead

### Final Solution
- Goals stored as `type: 'creation'` records in `amber_state` table
- Actual goal data in `metadata.type: 'weekly_goals'` with full structure
- API route updated to query: `eq('type', 'creation')` AND `eq('metadata->>type', 'weekly_goals')`
- Uses `SUPABASE_SERVICE_KEY` for full database access (not just anon key)
