## Technical Note: JT90 Section Rendering Pattern
```javascript
// CORRECT approach — render by sections:
const sections = [
  { startBar: 0, bars: 8, pattern: buildPattern({...}), chDecay: 0.10 },
  { startBar: 8, bars: 8, pattern: buildPattern({...}), chDecay: 0.12 },
  // ...
];
for (const sec of sections) {
  engine.setPattern(sec.pattern);
  const buffer = await engine.renderPattern({ bars: sec.bars, bpm, automation });
  barBuffers.push(buffer);
}
// DO NOT render bar-by-bar with setPattern each iteration — causes alternating state bug
```

## ⚠️ CRITICAL: JT10 ScriptProcessor Does NOT Work in OfflineAudioContext

**`ScriptProcessor.onaudioprocess` never fires in node-web-audio-api's `OfflineAudioContext`.**

This means:
- `playNote()` hangs forever (awaits `context.resume()` which never resolves on OfflineAudioContext)
- Manually creating `_scriptNode` via `createScriptProcessor` appears to work but `onaudioprocess` is never called — output is **silent**
- **Every JT10 render using `playNote` + `_scriptNode` routing produced ZERO synth audio** — only drums were audible

### The Fix: Use `JT10Engine.renderPattern()`
```javascript
const lead = new JT10Engine({ sampleRate: SR });
lead.setBpm(BPM);
lead.setParameter('cutoff', 0.5);  // etc
lead.masterVolume = 0.8;

// Pattern format: array of { note, gate, slide, accent }
const pattern = [
  { note: 'E3', gate: true,  slide: false, accent: false },
  { note: 'E3', gate: false, slide: false, accent: false },  // rest
  { note: 'G3', gate: true,  slide: false, accent: false },
  // ...
];
lead.setPattern(pattern);
const buffer = await lead.renderPattern({ bars: 8, sampleRate: SR });
const samples = buffer.getChannelData(0);  // Float32Array
```

Then mix drum + synth buffers manually:
```javascript
const mix = new Float32Array(Math.max(drumSamples.length, leadSamples.length));
for (let i = 0; i < mix.length; i++) {
  const d = i < drumSamples.length ? drumSamples[i] * 0.7 : 0;
  const l = i < leadSamples.length ? leadSamples[i] * 0.6 : 0;
  mix[i] = Math.tanh((d + l) * 1.1);
}
```

**NO OfflineAudioContext needed for basic renders.** Only use it if you need Web Audio effects (delay, reverb, convolver). For dry renders, just render each engine separately and sum the buffers.

## ✅ PROVEN: Combining JB01 + JT10 in Offline Render

**The only approach that works:**
1. Render JT10 via `renderPattern()` to Float32Array buffer (no AudioContext needed)
2. Create OfflineAudioContext for the final mix
3. JB01 works in OfflineAudioContext — use `connectOutput()` + `trigger()` via suspend/resume
4. Inject JT10 buffer as `AudioBufferSourceNode` into the same OfflineAudioContext
5. Add Web Audio effects (delay, reverb) as nodes in the OfflineAudioContext

```javascript
// Step 1: Render JT10 to buffer
const stab = new JT10Engine({ sampleRate: SR });
stab.setBpm(BPM);
stab.setParameter('cutoff', 0.30); // etc
stab.setPattern(pattern); // [{ note, gate, slide, accent }, ...]
const stabBuf = await stab.renderPattern({ bars: 16, sampleRate: SR });
const stabSamples = stabBuf.getChannelData(0);

// Step 2: OfflineAudioContext with JB01 + injected JT10
const actx = new OfflineAudioContext(2, totalSamples, SR);

// JB01 works directly in OfflineAudioContext
const drums = new JB01Engine({ context: actx });
drums.connectOutput(drumGain);
// Trigger via suspend/resume:
actx.suspend(t).then(() => { drums.trigger('kick', 0.85); actx.resume(); });

// Inject JT10 as AudioBufferSourceNode
const stabAudioBuf = actx.createBuffer(1, stabSamples.length, SR);
stabAudioBuf.getChannelData(0).set(stabSamples);
const stabSrc = actx.createBufferSource();
stabSrc.buffer = stabAudioBuf;
stabSrc.connect(delayNode); // route to effects
stabSrc.start(0);

const rendered = await actx.startRendering();
```

**Why this works:**
- JB01 uses Web Audio oscillators/noise internally — these work in OfflineAudioContext
- JT10 uses ScriptProcessor internally — this does NOT produce audio in OfflineAudioContext
- Rendering JT10 separately via `renderPattern()` bypasses ScriptProcessor entirely
- AudioBufferSourceNode playback works perfectly in OfflineAudioContext

**What does NOT work:**
- ❌ `JT10.playNote()` in OfflineAudioContext — hangs on `context.resume()`
- ❌ Manually creating `_scriptNode` — `onaudioprocess` fires but output is silently dropped
- ❌ Any approach using ScriptProcessor in OfflineAudioContext

## Fade Calculation Fix
When rendering by sections (not bars), can't use `barBuffers[0].length` for fade — sections have different lengths. Use:
```javascript
const samplesPerBar = Math.round(sampleRate * (60 / BPM) * 4);
const fadeLen = samplesPerBar * 2;
```
