### Drift — New Track (the big discovery)
- **Browser tab throttle discovery**: When you background the Pulse Grid tab, setInterval gets throttled to ~1/sec, but AudioContext keeps running. Notes trigger 10x slower, reverb fills gaps. Sounds incredibly organic — like a dramatic concert breakdown.
- Analyzed the screen recording audio: RMS drops ~60%, density drops from 95% to 80% during throttled section
- Built **drift.html** — intentionally reproduces this effect as an arrangement:
  - 64 bars at 118 BPM, 5 sections: RISE → PEAK → DISSOLVE → DRIFT → RETURN
  - DISSOLVE: note probability drops 100% → 12% over 16 bars
  - DRIFT: only ~12% of notes fire, 6s reverb at 90% wet, each note becomes an event
  - RETURN: notes gradually come back
- Published to `kochi.to/viz/drift.html`
- Uses probability-based note triggering (`Math.random() < prob`) — same pattern, just most gates skipped
- Filter cutoff also closes during drift (0.35 → 0.15) and reverb wet increases

### Key Technical Notes
- JT10 browser engine API: `playNote(note, vel, slide)`, `setParameter(id, value)`, `noteOff()`
- Engine uses scriptProcessor internally — fine for pure synthesis, NOT for sample-based (JT90)
- To add effects: force scriptNode creation with silent note, then `engine._scriptNode.disconnect()` and rewire to effect chain
- `engine.masterVolume` controls overall level

### Drift v2 changes (all three requested by Bart)
1. **Arp variations**: A natural minor scale (A B C D E F G). One note substitution per 4-bar block starting at bar 9. Not every block — some stay pure. Uses 7th (G3), 4th (D3), 2nd (B3), low 5th (E2). Subtle, like seasoning.
2. **Viz fade-to-black**: `vizBrightness` multiplier drops to 0.08 during drift. Canvas fade rate increases. Just a lone 2px breathing dot at center (sine-wave blink). Comes back on return.
3. **Reverb automation**: Starts almost dry (10% wet). Builds over 16 bars to 65%. Peak holds 65%. Dissolve pushes to 90%. Drift = 90% wet / 20% dry. Return pulls back.

### Technique: probability-based arp variations
- `variations` array indexed by 4-bar block (16 entries for 64 bars)
- Each entry is either `null` (pure) or `{ step, note, slide }` for one substitution
- `getArpForBar()` deep-copies base pattern and applies the sub
- Keep pure blocks between variations so changes register as events, not constant shifting
