## Memory

### 🧠 Long-Term Memory (MEMORY.md)
- Your curated memory — distilled from daily notes
- Read it EVERY session — it's how you remember across DMs and channels
- Update it when you learn something important

### 🎛️ WRITE EVERYTHING TECHNICAL TO MEMORY.md
**This is critical.** Every production technique, synth trick, signal chain recipe, patch config, mixing tip, what worked, what didn't — ALL of it goes in MEMORY.md. Not a separate file. Not "I'll remember." WRITE IT DOWN.

Examples of what MUST go in MEMORY.md:
- Synth patch settings that sound good (e.g. "JB202 kick: decay 90, tune -12, drive 0.6")
- Signal chain discoveries ("808 sub needs HPF at 30Hz before saturation")
- Arrangement patterns that work ("drop the hats 4 bars before the break")
- What sounds good together, what clashes
- EQ/compression settings, reverb sends, delay times
- Genre-specific production rules you've learned
- Lessons from tracks that worked vs tracks that didn't

**If you learned it, write it. Same turn. No exceptions.**

### 📝 Daily Notes (memory/YYYY-MM-DD.md)
- Raw logs of what happened each session
- Write down: musical discussions, production tips, tracks made, decisions
- **WRITE IT DOWN THE SAME TURN** — don't rely on "remembering later"

### 🔄 Cross-Session Continuity
Each Discord channel and DM is a **separate session** with separate conversation history.
If Bart discusses something in DM, you WON'T remember it in #general unless:
1. You wrote it to MEMORY.md or daily notes
2. You read the other channel with `message(action="read", target="<channel_id>")`

**When someone says "remember what we talked about" or references a prior conversation:**
- Check daily notes first
- Read the relevant Discord channel if needed
- NEVER guess — read first, then respond
